1. GIT
1.1 Structure of local (master branch) to github (hosting - remote branch) and moving files to github

    Local (master branch)
    - git clone link from github
    - git status and git init
    - edit a file
    - save locally (working directory)

    Local to staging before pushing to github
    - git add to place file in staging area (including git add .). Note: still local only
    - git commit them (all added files) to the master branch which we have locally

    Staging to github
    - Check for remote branch (will have if cloned from github) with command git remote -v [v for view].
      This command returns the url of the remote repo.
    - By convention the remote branch is named "origin"
    - Note: when creating a repository on github you will have the option of cloning a local directory.
      But we created on github first and then cloned to local in the setup phase.
    - Note: git remote rename and git remote remove to rename and remove the remote repository.
    - Command: git push -u origin main

2. Virtual Environments

    Copy of the base installation which allows versions of libraries to be installed as required
    it is created (standard venv) and activeated (switching from base to virtual).

    Activation involves updating the PATH variable with the correct environment for the python
    binary being executed.

2.1 Creation commands

    which python
    python --version
    <python version/path> -m venv <your environment name>


2.2 Activation commands non-windows

    source <path-to-env>/bin/activate

    3. Running Jupyter Notebooks
    ============================

    From a command prompt:
    1. Navigate to your project root folder.
    2. Activate your virtual environment
    3. run Jupyter notebooks: jupyter notebook .
    4. If a web browser does not automatically open up, find the URL in the command prompt window (output when Jupyter starts up), and paste that URL into your browser.

    You can create Jupyter notebooks anywhere in your project folder - it does not have to be in the root of your project folder, you can create sub-folders and organize your notebooks however you want - just like you wold organize Excel or Word documents.

    Keyboard Shortcuts
    ------------------
    - execute a cell: Shift+Enter
    - select current cell: Esc (or click on cell margin)
    - delete selected cell: DD (type D twice in succession)
    - change selected cell to Markup: M
    - change selected cell to Code: Y
    - save: S
    - insert cell above current selected cell: B
    - insert cell after currently selected cell: A

    Don't forget to re-run your Jupyter notebook when you re-load it later - this will just re-execute all the cells (but it will stop processing the cells if it encounters an error).

    Jupyter Notebook documentation: https://jupyter-notebook.readthedocs.io/en/stable/
